# dev-issue: 기술적 습득 및 해결 과제

## 1. Dirty Checking vs Atomic Query
- **습득**: JPA의 변경 감지는 편리하지만, 단순 조회수 증가 로직에서는 불필요한 SELECT 후 UPDATE가 발생한다는 점을 알게 되었습니다. 이를 해결하기 위해 DB 레벨에서 즉시 연산하는 Atomic Query 방식의 효율성을 학습했습니다.

## 2. @Modifying 사용 시 트랜잭션 필수 요구 사항
- **이슈**: Repository에 `@Modifying`을 사용한 조회수 증가 로직을 추가한 후, Service에서 상세 조회 시 해당 메서드를 호출하자 런타임 에러(`TransactionRequiredException`)가 발생했습니다.
- **원인**: `@Modifying`과 `@Query`를 통한 직접적인 데이터 수정은 영속성 컨텍스트를 거치지 않고 DB에 직접 쿼리를 날리는 작업이므로, 데이터 정합성과 안정성을 위해 반드시 활성화된 트랜잭션이 필요함을 확인했습니다.
- **해결**: Service 레이어의 상세 조회 메서드에 `@Transactional` 어노테이션을 추가하여 읽기뿐만 아니라 조회수 업데이트 작업이 하나의 트랜잭션 범위 안에서 안전하게 수행되도록 수정했습니다.

## 3. XSS(Cross-Site Scripting)와 보안 전략
- **습득**: 공격자가 스크립트를 심어 토큰을 탈취하는 XSS의 위험성을 이해했습니다. 이를 방어하기 위해 자바스크립트가 접근할 수 없는 `HttpOnly` 쿠키에 중요 정보(RT)를 저장하는 전략이 왜 필수적인지 체득했습니다.

## 4. HTTPS의 실질적 보안 이점
- **습득**: 단순한 암호화를 넘어, 데이터 무결성 보장과 Secure 쿠키 사용을 위해 HTTPS가 인프라의 기본임을 새로 알게 되었습니다.

## 5. Slice를 통한 무한 스크롤 최적화
- **습득**: `Page` 인터페이스가 대량 데이터에서 왜 성능 병목이 되는지(Count 쿼리) 분석했고, 무한 스크롤 UI에는 `Slice`가 훨씬 적합한 기술임을 확인했습니다.
