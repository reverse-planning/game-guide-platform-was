# ADR-006: 목록 조회 최적화 전략 (Slice & Excerpt)

## 상태
Accepted

## 컨텍스트
대량의 게시글 목록을 효율적으로 전달하고, 무한 스크롤 환경에서의 성능 저하를 방지해야 했습니다.

## 결정
1. **Slice**: 무한 스크롤 구현을 위해 `Page` 대신 `Slice`를 사용합니다.
2. **Excerpt**: DTO 레이어에서 본문을 100자로 요약하여 제공합니다.

## 근거
- **Slice vs Page**: `Page`는 전체 카운트 쿼리를 실행하여 부하가 크지만, `Slice`는 다음 페이지 존재 여부만 확인(`limit + 1`)하여 성능상 유리합니다.
- **Excerpt**: 전체 본문을 전송하는 것보다 네트워크 비용을 절감할 수 있으며, `Math.min`을 활용해 안전하게 데이터를 가공합니다.

## 결과
- 불필요한 카운트 쿼리 제거로 응답 속도가 향상되었으며 네트워크 대역폭을 절약했습니다.
