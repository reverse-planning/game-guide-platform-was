# ADR-001: 사용자 인증 방식 결정 (Session vs JWT)

## 상태
Accepted

## 컨텍스트
사용자 로그인 상태를 유지하기 위해 서버 측에서 세션을 관리할지, 클라이언트 측에서 토큰을 소유하는 JWT 방식을 사용할지에 대한 결정이 필요했습니다. 서비스의 확장성과 서버 리소스 효율성을 고려해야 했습니다.

## 결정
무상태성(Stateless)을 유지할 수 있는 **JWT(JSON Web Token)** 방식을 채택했습니다.

## 근거
- **확장성**: 세션 방식은 서버 메모리에 상태를 저장하므로 서버 대수가 늘어날 때 세션 동기화 비용이 발생하지만, JWT는 서버가 상태를 가질 필요가 없어 수평 확장(Scale-out)에 유리합니다.
- **비용**: 서버의 메모리 부하를 줄이고 클라이언트가 인증 정보를 직접 관리하게 함으로써 인프라 운영 비용을 절감할 수 있습니다.

## 결과
- 모든 API 요청 헤더에 토큰을 포함해야 하며, 서버는 이를 검증하는 필터가 필요합니다.
- (여담) 이후 벤치마킹 시, 세션 기반의 서버 메모리 점유율과 JWT 기반의 CPU 연산량 차이를 비교군으로 활용할 예정입니다.
