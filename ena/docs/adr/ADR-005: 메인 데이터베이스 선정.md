# ADR-005: 메인 데이터베이스 선정 (MySQL)

## 상태
Accepted

## 컨텍스트
서비스의 게시글, 유저, 게임 정보를 저장하기 위해 어떤 유형의 데이터베이스를 사용할지 결정해야 했습니다. 데이터 간의 복잡한 연관 관계를 관리하고 트랜잭션을 보장해야 합니다.

## 결정
관계형 데이터베이스(RDBMS)인 **MySQL**을 메인 데이터베이스로 채택했습니다.

## 근거
- **데이터 무결성**: 유저-게임-게시글로 이어지는 참조 관계를 외래 키(Foreign Key)를 통해 엄격하게 관리할 수 있습니다.
- **표준성 및 신뢰도**: 전 세계적으로 가장 많이 쓰이는 표준 RDB로서, 이후 Redis 도입 전후의 성능 차이를 증명할 때 신뢰할 수 있는 성능 지표(Baseline)를 제공합니다.

## 결과
- JPA 엔티티 매핑을 통해 객체 지향적인 데이터 모델링을 수행합니다.
- (여담) 향후 MySQL의 디스크 I/O 병목 현상을 측정하고 Redis 캐싱으로 이를 어떻게 극복하는지 수치화할 예정입니다.
