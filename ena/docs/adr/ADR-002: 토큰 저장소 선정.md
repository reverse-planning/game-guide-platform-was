# ADR-003: 토큰 보안 및 저장 전략 (AT/RT 저장 위치)

## 상태
Accepted

## 컨텍스트
JWT 사용 시 Access Token(AT)과 Refresh Token(RT)을 클라이언트 어디에 저장하느냐에 따라 XSS 및 CSRF 공격 노출 정도가 달라집니다.

## 결정
- **Access Token**: Authorization Header(Bearer)에 담아 전송하며, 프론트엔드 메모리 상에서 관리합니다.
- **Refresh Token**: 브라우저의 **HttpOnly, Secure 쿠키(Cookie)**에 저장합니다.

## 근거
- **HttpOnly Cookie**: 자바스크립트가 쿠키에 접근할 수 없게 차단하여 XSS 공격을 통한 토큰 탈취를 원천적으로 방지합니다.
- **Secure**: HTTPS 프로토콜에서만 토큰이 전송되도록 강제하여 통신 보안을 강화합니다.

## 결과
- 보안성을 극대화하면서도 자동 로그인(재발급) 기능을 안정적으로 구현했습니다.
- CORS 설정 시 `withCredentials: true` 및 특정 도메인 허용 설정이 필수적입니다.
